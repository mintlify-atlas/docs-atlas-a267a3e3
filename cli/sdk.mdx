## Usage

```bash
tsgonest sdk [flags]
```

Generates a fully-typed TypeScript SDK from your OpenAPI 3.2 specification. The SDK includes:

- Type-safe client with auto-complete
- One function per API endpoint
- Request/response types
- Error handling
- Support for global prefix and versioning

## Flags

<ParamField path="--input" type="string" required>
  Path to OpenAPI JSON file
  
  ```bash
  tsgonest sdk --input dist/openapi.json
  tsgonest sdk --input api-spec.json
  ```
  
  Can be configured in `tsgonest.config.ts`:
  
  ```typescript
  export default defineConfig({
    sdk: {
      input: "dist/openapi.json",  // Defaults to openapi.output
    },
  });
  ```
</ParamField>

<ParamField path="--output" type="string" default="./sdk">
  Output directory for generated SDK
  
  ```bash
  tsgonest sdk --input dist/openapi.json --output ./client
  tsgonest sdk --input api.json --output packages/api-client
  ```
  
  Can be configured in `tsgonest.config.ts`:
  
  ```typescript
  export default defineConfig({
    sdk: {
      output: "./client",
    },
  });
  ```
</ParamField>

<ParamField path="--config" type="string">
  Path to tsgonest config file
  
  Loads SDK options (globalPrefix, versioning) from config.
  
  ```bash
  tsgonest sdk --config tsgonest.config.ts
  ```
</ParamField>

## Configuration

### Config File

Configure SDK generation in `tsgonest.config.ts`:

```typescript tsgonest.config.ts
import { defineConfig } from "tsgonest";

export default defineConfig({
  openapi: {
    output: "dist/openapi.json",
  },
  sdk: {
    output: "./sdk",
    // input defaults to openapi.output
  },
  nestjs: {
    globalPrefix: "api",
    versioning: {
      type: "URI",
      defaultVersion: "1",
      prefix: "v",
    },
  },
});
```

With this config, running `tsgonest sdk` (no flags) will:
1. Use `dist/openapi.json` as input
2. Generate SDK to `./sdk`
3. Apply global prefix and versioning transformations

### NestJS Settings

The SDK generator respects NestJS configuration:

<ParamField path="nestjs.globalPrefix" type="string">
  Global API prefix
  
  If your NestJS app uses `app.setGlobalPrefix("api")`, configure it here so the SDK uses correct paths:
  
  ```typescript
  nestjs: {
    globalPrefix: "api",
  }
  ```
  
  Generated SDK will call:
  - `/api/users` instead of `/users`
  - `/api/products` instead of `/products`
</ParamField>

<ParamField path="nestjs.versioning" type="object">
  API versioning configuration
  
  ```typescript
  nestjs: {
    versioning: {
      type: "URI",              // URI, HEADER, MEDIA_TYPE
      defaultVersion: "1",
      prefix: "v",              // Default: "v"
    },
  }
  ```
  
  With URI versioning, SDK calls:
  - `/api/v1/users` (default version)
  - `/api/v2/users` (if endpoint specifies version 2)
</ParamField>

## Generated SDK Structure

The SDK is generated with the following structure:

```
sdk/
├── index.ts          # Main client export
├── client.ts         # HTTP client with fetch wrapper
├── types.ts          # Request/response types
└── endpoints/
    ├── users.ts      # Users endpoints
    ├── products.ts   # Products endpoints
    └── index.ts      # Barrel export
```

### Client

The main client class:

```typescript sdk/index.ts
export class ApiClient {
  constructor(private baseUrl: string, private options?: RequestInit) {}
  
  users = {
    getUsers: (params?: GetUsersParams) => Promise<User[]>,
    getUser: (id: string) => Promise<User>,
    createUser: (body: CreateUserDto) => Promise<User>,
    updateUser: (id: string, body: UpdateUserDto) => Promise<User>,
    deleteUser: (id: string) => Promise<void>,
  };
  
  products = {
    getProducts: () => Promise<Product[]>,
    // ...
  };
}
```

### Types

All DTOs and schemas are exported:

```typescript sdk/types.ts
export interface User {
  id: string;
  email: string;
  name: string;
}

export interface CreateUserDto {
  email: string;
  name: string;
  password: string;
}

export interface GetUsersParams {
  page?: number;
  limit?: number;
  search?: string;
}
```

## Usage Examples

### Generate SDK

```bash
# Using config file:
tsgonest sdk

# With explicit paths:
tsgonest sdk --input dist/openapi.json --output ./client
```

### Use Generated SDK

```typescript
import { ApiClient } from './sdk';

const client = new ApiClient('http://localhost:3000');

// Get all users (with autocomplete!)
const users = await client.users.getUsers({
  page: 1,
  limit: 10,
  search: 'john',
});

// Create user
const newUser = await client.users.createUser({
  email: 'john@example.com',
  name: 'John Doe',
  password: 'secret123',
});

// Update user
const updated = await client.users.updateUser(newUser.id, {
  name: 'Jane Doe',
});
```

### Custom Base URL

```typescript
const client = new ApiClient(process.env.API_URL || 'http://localhost:3000');
```

### Authentication

```typescript
const client = new ApiClient('http://localhost:3000', {
  headers: {
    'Authorization': `Bearer ${token}`,
  },
});
```

### Error Handling

```typescript
try {
  const user = await client.users.getUser('invalid-id');
} catch (error) {
  if (error instanceof ApiError) {
    console.error('API Error:', error.status, error.message);
    console.error('Response:', error.body);
  } else {
    console.error('Network Error:', error);
  }
}
```

## Integration with Build

The SDK is automatically generated during `tsgonest build` if `sdk.output` is configured:

```typescript tsgonest.config.ts
export default defineConfig({
  openapi: {
    output: "dist/openapi.json",
  },
  sdk: {
    output: "./sdk",
  },
});
```

```bash
$ tsgonest build
emitted 10 file(s)
generated OpenAPI document: dist/openapi.json
generated SDK: ./sdk
```

### Incremental SDK Generation

SDK generation is skipped if the OpenAPI spec hasn't changed:

```bash
$ tsgonest build
emitted 0 file(s)
SDK up to date, skipping generation
```

The hash of the input file is cached in `<sdk-output>/.sdk-hash`.

## Monorepo Setup

For monorepos, generate the SDK into a shared package:

```typescript apps/api/tsgonest.config.ts
export default defineConfig({
  openapi: {
    output: "dist/openapi.json",
  },
  sdk: {
    output: "../../packages/api-client/src",
  },
});
```

```json packages/api-client/package.json
{
  "name": "@myapp/api-client",
  "version": "1.0.0",
  "main": "src/index.ts",
  "exports": {
    ".": "./src/index.ts"
  }
}
```

Other apps can import the client:

```typescript apps/web/src/lib/api.ts
import { ApiClient } from '@myapp/api-client';

export const api = new ApiClient(process.env.NEXT_PUBLIC_API_URL);
```

## Advanced Configuration

### Custom Fetch Options

Pass default options to all requests:

```typescript
const client = new ApiClient('http://localhost:3000', {
  headers: {
    'Content-Type': 'application/json',
    'X-API-Key': process.env.API_KEY,
  },
  credentials: 'include',  // Send cookies
});
```

### Interceptors

The generated client supports request/response interceptors:

```typescript
import { ApiClient } from './sdk';

class AuthenticatedClient extends ApiClient {
  private token: string | null = null;
  
  setToken(token: string) {
    this.token = token;
  }
  
  protected async request(path: string, options?: RequestInit) {
    const headers = new Headers(options?.headers);
    if (this.token) {
      headers.set('Authorization', `Bearer ${this.token}`);
    }
    
    return super.request(path, { ...options, headers });
  }
}

const client = new AuthenticatedClient('http://localhost:3000');
client.setToken('your-token');
```

## Exit Codes

<ResponseField name="0" type="Success">
  SDK generated successfully
  
  ```bash Terminal
  $ tsgonest sdk --input dist/openapi.json
  SDK generated at ./sdk
  $ echo $?
  0
  ```
</ResponseField>

<ResponseField name="1" type="Error">

  ```bash Terminal
  $ tsgonest sdk --input missing.json

  ```
</ResponseField>

## Troubleshooting

### "--input is required"

Either pass `--input` or configure `sdk.input` or `openapi.output` in config:

```typescript
export default defineConfig({
  openapi: {
    output: "dist/openapi.json",
  },
  sdk: {
    // input defaults to openapi.output
  },
});
```

### "could not read input file"

Ensure the OpenAPI file exists:

```bash
ls -la dist/openapi.json
```

Run build first:

```bash
tsgonest build
tsgonest sdk
```

### Generated SDK has type errors

Regenerate the SDK after fixing OpenAPI:

```bash
rm -rf ./sdk
tsgonest build
```

If the issue persists, check your OpenAPI spec:

```bash
cat dist/openapi.json | jq .
```

### SDK doesn't match API behavior

Verify that `globalPrefix` and `versioning` in config match your NestJS app:

```typescript main.ts
const app = await NestFactory.create(AppModule);
app.setGlobalPrefix('api');  // Must match config
app.enableVersioning({
  type: VersioningType.URI,
  defaultVersion: '1',
  prefix: 'v',               // Must match config
});
```
