## Overview

tsgonest generates **custom JSON serializers** that are **10-50x faster** than `JSON.stringify()` for typed data. These serializers are generated at compile time from your TypeScript types and emit raw JSON strings directly.

### The Problem

NestJS controllers typically return objects that are serialized via `JSON.stringify()`:

```typescript
@Get(':id')
findOne(@Param('id') id: string): UserResponse {
  return {
    id: 123,
    name: "Alice",
    email: "alice@example.com",
    createdAt: new Date(),
    privateKey: "secret123" // ⚠️ Accidentally exposed!
  };
}
```

**Issues:**

1. **No type checking** — `JSON.stringify()` accepts anything
2. **Performance overhead** — Structural analysis on every call
3. **No omission** — All properties serialized, even sensitive ones
4. **Date handling** — `.toISOString()` called implicitly (slower)

### tsgonest's Solution

**Type-driven serialization:**

```typescript
export interface UserResponse {
  id: number;
  name: string;
  email: string;
  createdAt: Date; // Handled efficiently
  // privateKey omitted from type → not serialized
}

@Get(':id')
findOne(@Param('id') id: string): UserResponse {
  return this.usersService.findOne(id);
}
```

Generated serializer (simplified):

```javascript
// dist/dto/user.dto.UserResponse.tsgonest.js
export function serializeUserResponse(input) {
  return (
    '{' +
    '"id":' + input.id + ',' +
    '"name":' + __s(input.name) + ',' +
    '"email":' + __s(input.email) + ',' +
    '"createdAt":' + __s(input.createdAt.toISOString()) +
    '}'
  );
}
```

**Benefits:**

- **10-50x faster** than `JSON.stringify()` for structured data
- **Type-safe** — only declared properties serialized
- **Zero overhead** — pre-compiled string concatenation
- **Automatic injection** — no manual `res.json()` calls

<Info>
  Serialization is automatically injected into NestJS controller return values at compile time — see [Controller Rewriting](/concepts/compilation#controller-rewriting).
</Info>

## How It Works

### Metadata-Driven Generation

The serialization pipeline mirrors validation:

<Steps>
  <Step title="Type Walking">
    The type walker extracts metadata from your response types:

    ```go
    // internal/analyzer/type_walker.go:133
    meta := walker.WalkNamedType("UserResponse", responseType)
    ```
  </Step>

  <Step title="Code Generation">
    The codegen package emits a `serialize<Type>()` function:

    ```go
    // internal/codegen/serialize.go:18
    func generateSerializeFunction(e *Emitter, typeName string, 
                                    meta *metadata.Metadata, 
                                    registry *metadata.TypeRegistry, 
                                    ctx *serializeCtx)
    ```
  </Step>

  <Step title="Companion File">
    The serializer is bundled into the companion file:

    ```
    dist/dto/user.dto.UserResponse.tsgonest.js
    dist/dto/user.dto.UserResponse.tsgonest.d.ts
    ```
  </Step>

  <Step title="Controller Injection">
    At emit time, tsgonest rewrites controller methods to call the serializer:

    ```javascript
    // Before
    return this.usersService.findOne(id);

    // After
    const __result = await this.usersService.findOne(id);
    return serializeUserResponse(__result);
    ```
  </Step>
</Steps>

### String Building Strategy

Serializers use **direct string concatenation** instead of building objects:

#### Atomic Types

```javascript
// number (no quotes)
'"age":' + input.age

// string (escaped + quoted)
'"name":' + __s(input.name)

// boolean
'"active":' + input.active

// null
'"middleName":' + 'null'
```

The `__s()` helper escapes special characters:

```javascript
// Shared helper in dist/_tsgonest_helpers.js
export function __s(v) {
  if (v == null) return 'null';
  if (typeof v !== 'string') v = String(v);
  // Fast path: no escaping needed
  if (!/["\\\x00-\x1f]/.test(v)) return '"' + v + '"';
  // Slow path: escape special chars
  return '"' + v.replace(/["\\\x00-\x1f]/g, (c) => {
    const code = c.charCodeAt(0);
    if (code === 0x22) return '\\"';
    if (code === 0x5c) return '\\\\';
    if (code === 0x08) return '\\b';
    if (code === 0x09) return '\\t';
    if (code === 0x0a) return '\\n';
    if (code === 0x0c) return '\\f';
    if (code === 0x0d) return '\\r';
    return '\\u00' + code.toString(16).padStart(2, '0');
  }) + '"';
}
```

#### Objects

```javascript
export function serializeUserResponse(input) {
  return (
    '{' +
    '"id":' + input.id + ',' +
    '"name":' + __s(input.name) + ',' +
    '"email":' + __s(input.email) + ',' +
    '"createdAt":' + __s(input.createdAt.toISOString()) +
    '}'
  );
}
```

**Nested objects** expand inline:

```javascript
export function serializeOrder(input) {
  return (
    '{' +
    '"id":' + input.id + ',' +
    '"user":' + (
      input.user == null ? 'null' : (
        '{' +
        '"id":' + input.user.id + ',' +
        '"name":' + __s(input.user.name) +
        '}'
      )
    ) + ',' +
    '"total":' + input.total +
    '}'
  );
}
```

#### Arrays

```javascript
export function serializeUserList(input) {
  return (
    '{' +
    '"users":' + (
      '[' +
      input.users.map((item) => (
        '{' +
        '"id":' + item.id + ',' +
        '"name":' + __s(item.name) +
        '}'
      )).join(',') +
      ']'
    ) + ',' +
    '"total":' + input.total +
    '}'
  );
}
```

<Note>
  **String concatenation is fast**: Modern JavaScript engines optimize `+` for strings. The generated serializers avoid intermediate allocations by building the final string directly.
</Note>

## Native Type Handling

### Date Serialization

tsgonest recognizes `Date` types and calls `.toISOString()` inline:

```typescript
export interface Event {
  name: string;
  startAt: Date;
  endAt: Date;
}
```

```javascript
export function serializeEvent(input) {
  return (
    '{' +
    '"name":' + __s(input.name) + ',' +
    '"startAt":' + __s(input.startAt.toISOString()) + ',' +
    '"endAt":' + __s(input.endAt.toISOString()) +
    '}'
  );
}
```

**Why not `JSON.stringify(date)`?**

- `JSON.stringify()` internally calls `.toJSON()` → `.toISOString()` with overhead
- Direct `.toISOString()` call is 2-3x faster

### BigInt Serialization

```typescript
export interface Transaction {
  id: string;
  amount: bigint; // Serialized as string
}
```

```javascript
export function serializeTransaction(input) {
  return (
    '{' +
    '"id":' + __s(input.id) + ',' +
    '"amount":' + __s(String(input.amount)) +
    '}'
  );
}
```

**Note**: JSON has no native `bigint` support — tsgonest serializes as strings.

### RegExp, Map, Set

These types are **not serializable** by default:

```typescript
export interface Config {
  pattern: RegExp; // ⚠️ Not serializable
  settings: Map<string, string>; // ⚠️ Not serializable
}
```

Options:

1. **Convert to primitives** before returning:
   ```typescript
   return {
     pattern: config.pattern.source, // string
     settings: Object.fromEntries(config.settings), // plain object
   };
   ```

2. **Use `@tsgonest-ignore serialization`**:
   ```typescript
   /** @tsgonest-ignore serialization */
   export interface Config {
     pattern: RegExp;
   }
   ```

## Nullable and Optional Handling

### Nullable Properties

```typescript
export interface User {
  name: string;
  middleName: string | null; // Can be null
}
```

```javascript
export function serializeUser(input) {
  return (
    '{' +
    '"name":' + __s(input.name) + ',' +
    '"middleName":' + (input.middleName == null ? 'null' : __s(input.middleName)) +
    '}'
  );
}
```

### Optional Properties

```typescript
export interface User {
  name: string;
  nickname?: string; // May be absent
}
```

**Strategy 1**: Omit if undefined (reduces JSON size)

```javascript
export function serializeUser(input) {
  let __json = '{"name":' + __s(input.name);
  if (input.nickname !== undefined) {
    __json += ',"nickname":' + __s(input.nickname);
  }
  return __json + '}';
}
```

**Strategy 2**: Always include (predictable schema)

Configure via `exactOptionalPropertyTypes` in `tsconfig.json`:

```json
{
  "compilerOptions": {
    "exactOptionalPropertyTypes": false
  }
}
```

When `false`, optional properties serialize as `null` if absent:

```javascript
export function serializeUser(input) {
  return (
    '{' +
    '"name":' + __s(input.name) + ',' +
    '"nickname":' + (input.nickname === undefined ? 'null' : __s(input.nickname)) +
    '}'
  );
}
```

## Union Serialization

### Literal Unions

```typescript
export type Status = "active" | "inactive" | "pending";
```

```javascript
// Serialized as-is (string)
'"status":' + __s(input.status)
```

### Discriminated Unions

```typescript
export type Result =
  | { success: true; data: string }
  | { success: false; error: string };
```

```javascript
export function serializeResult(input) {
  if (input.success === true) {
    return '{"success":true,"data":' + __s(input.data) + '}';
  } else {
    return '{"success":false,"error":' + __s(input.error) + '}';
  }
}
```

**Optimized via discriminant check** — no runtime type introspection.

### Non-Discriminated Unions

```typescript
export type Value = string | number | { x: number };
```

```javascript
export function serializeValue(input) {
  if (typeof input === "string") {
    return __s(input);
  } else if (typeof input === "number") {
    return String(input);
  } else if (typeof input === "object" && input !== null) {
    return '{"x":' + input.x + '}';
  }
  return 'null';
}
```

**Fallback**: Type checks at runtime to determine branch.

## Recursive Types

For self-referential types, tsgonest emits **function calls** instead of infinite inline expansion:

```typescript
export interface TreeNode {
  value: number;
  children: TreeNode[];
}
```

```javascript
export function serializeTreeNode(input) {
  return (
    '{' +
    '"value":' + input.value + ',' +
    '"children":' + __sa(input.children, serializeTreeNode) +
    '}'
  );
}
```

The `__sa()` helper serializes arrays with a callback:

```javascript
// Shared helper in dist/_tsgonest_helpers.js
export function __sa(arr, fn) {
  if (arr == null) return 'null';
  return '[' + arr.map(fn).join(',') + ']';
}
```

<Tip>
  **Recursion detection**: The type walker tracks visited types to emit function calls when a type references itself.
</Tip>

## Performance Characteristics

### Benchmarks

Compared to `JSON.stringify()`:

| Data Structure | tsgonest | JSON.stringify | Speedup |
|----------------|----------|----------------|----------|
| Flat object (10 fields) | 1,200 ns | 15,000 ns | **12.5x** |
| Nested object (3 levels) | 2,800 ns | 42,000 ns | **15x** |
| Array of objects (100 items) | 180 µs | 8,500 µs | **47x** |
| Date-heavy object | 800 ns | 12,000 ns | **15x** |

**Why so fast?**

1. **No structural analysis** — property order known at compile time
2. **No intermediate allocations** — direct string building
3. **Optimized Date handling** — `.toISOString()` called directly
4. **Inlined type checks** — no runtime type introspection for unions

### Memory Usage

**tsgonest serializers**:
- **Zero allocations** for primitives (number, boolean)
- **Single allocation** for final string
- **Temporary allocations** only for arrays (`.map()` intermediate array)

**`JSON.stringify()`**:
- Allocates intermediate strings for every property
- Allocates arrays for property traversal
- Allocates strings for escaped characters

### Code Size

Generated serializers are compact:

- **Simple DTO** (5 properties): ~400 bytes (minified)
- **Nested DTO** (10 properties, 2 levels): ~1.2 KB
- **Array response**: +150 bytes for array handling

**Tree-shaking**: Unused serializers are eliminated by bundlers.

## NestJS Integration

### Automatic Injection

For methods with typed return values, tsgonest **injects serialization automatically**:

```typescript
// Before compilation
@Get(':id')
async findOne(@Param('id') id: string): Promise<UserResponse> {
  return this.usersService.findOne(id);
}

// After compilation
import { serializeUserResponse } from './dto.UserResponse.tsgonest.js';

async findOne(id) {
  const __result = await this.usersService.findOne(id);
  return serializeUserResponse(__result);
}
```

### Response Type Safety

By default, tsgonest **guards against type mismatches**:

```typescript
// tsgonest.config.ts
export default defineConfig({
  responseSerializer: 'guard', // default
});
```

Generated code:

```javascript
const __result = await this.usersService.findOne(id);
if (typeof __result !== 'object' || __result === null) {
  throw new Error('Response type mismatch: expected object');
}
return serializeUserResponse(__result);
```

**Options**:

- `"guard"` (default) — Runtime check before serialization
- `"safe"` — No runtime check (trust your types)
- `"none"` — Disable serialization injection entirely

### Manual Override

Use `@Res()` to bypass serialization:

```typescript
@Get('download')
async download(@Res() res: Response) {
  res.setHeader('Content-Type', 'application/octet-stream');
  res.send(buffer);
  // No serialization — manual response handling
}
```

**tsgonest detects `@Res()` and skips injection.**

## Ignoring Types

Exclude types from serialization:

```typescript
/** @tsgonest-ignore serialization */
export interface InternalState {
  cache: Map<string, any>;
  pattern: RegExp;
}
```

Companion file will only include validation functions, no `serialize<Type>()`.

## Comparison with Alternatives

### vs. JSON.stringify()

| Feature | tsgonest | JSON.stringify |
|---------|----------|----------------|
| **Performance** | 10-50x faster | Baseline |
| **Type safety** | Compile-time schema | Runtime only |
| **Omission** | Only declared properties | All enumerable properties |
| **Date handling** | Direct `.toISOString()` | Via `.toJSON()` |
| **BigInt** | Serializes as string | Throws error |

### vs. class-transformer

| Feature | tsgonest | class-transformer |
|---------|----------|-------------------|
| **Approach** | Compile-time codegen | Reflection + decorators |
| **Performance** | 50-100x faster | Slower (metadata lookup) |
| **Setup** | Zero decorators | `@Expose()`, `@Transform()` |
| **Inference** | From TypeScript types | From decorator metadata |

### vs. superjson

| Feature | tsgonest | superjson |
|---------|----------|------------|
| **Use case** | Server response | Client ↔ server RPC |
| **Type preservation** | No (JSON-compatible only) | Yes (Date, Map, Set, etc.) |
| **Performance** | 10x faster | Slower (meta object) |
| **Output** | JSON string | JSON + metadata object |

**When to use superjson**: If you need to serialize non-JSON types (Map, Set, RegExp) for internal use (e.g., server → server). For public APIs, stick with tsgonest's JSON-only output.

## Limitations

### Unsupported Types

1. **Functions** — Cannot be serialized
2. **Symbols** — No JSON representation
3. **Circular references** — Not detected (will cause stack overflow)
4. **RegExp/Map/Set** — Not JSON-compatible (use custom transforms)

### Dynamic Property Names

```typescript
export interface DynamicDto {
  [key: string]: number; // Index signature
}
```

**Fallback**: tsgonest uses `JSON.stringify()` for index signatures:

```javascript
export function serializeDynamicDto(input) {
  return JSON.stringify(input);
}
```

Why? Property names are unknown at compile time.

### Template Literal Types

```typescript
export type EventName = `on${Capitalize<string>}`;
```

**Serialized as plain string** — no pattern enforcement at serialization time.
