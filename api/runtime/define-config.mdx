## Overview

The `defineConfig` function provides type-safe autocomplete and validation for your tsgonest configuration file. It's a simple identity function that returns the config object unchanged, but enables TypeScript to provide IntelliSense for all available options.

## Import

```typescript
import { defineConfig } from '@tsgonest/runtime';
```

## Usage Example

### Basic Configuration

```typescript tsgonest.config.ts
import { defineConfig } from '@tsgonest/runtime';

export default defineConfig({
  controllers: {
    include: ['src/**/*.controller.ts'],
  },
  transforms: {
    validation: true,
    serialization: true,
  },
  openapi: {
    output: 'dist/openapi.json',
    title: 'My API',
    version: '1.0.0',
  },
});
```

### Full Configuration

```typescript tsgonest.config.ts
import { defineConfig } from '@tsgonest/runtime';

export default defineConfig({
  controllers: {
    include: ['src/**/*.controller.ts'],
    exclude: ['**/*.spec.ts'],
  },
  transforms: {
    validation: true,
    serialization: true,
    standardSchema: true,
    responseSerializer: 'guard',
    include: ['src/**/*.dto.ts'],
    exclude: ['Legacy*'],
  },
  openapi: {
    output: 'dist/openapi.json',
    title: 'My API',
    version: '1.0.0',
    description: 'API for my application',
    contact: {
      name: 'API Support',
      email: 'support@example.com',
      url: 'https://example.com/support',
    },
    license: {
      name: 'MIT',
      url: 'https://opensource.org/licenses/MIT',
    },
    servers: [
      { url: 'https://api.example.com', description: 'Production' },
      { url: 'http://localhost:3000', description: 'Development' },
    ],
    securitySchemes: {
      bearer: {
        type: 'http',
        scheme: 'bearer',
        bearerFormat: 'JWT',
      },
    },
    security: [{ bearer: [] }],
  },
  sdk: {
    output: './sdk',
  },
  nestjs: {
    globalPrefix: 'api',
    versioning: {
      type: 'URI',
      defaultVersion: '1',
      prefix: 'v',
    },
  },
  entryFile: 'main',
  sourceRoot: 'src',
  deleteOutDir: true,
});
```

## Notes

- Configuration files can be named `tsgonest.config.ts` (recommended) or `tsgonest.config.json`
- TypeScript config files are evaluated via Node.js for dynamic configuration
- Use `defineConfig` to get full IntelliSense support in your IDE
- All configuration options are optional with sensible defaults
